## ✦ About

> *"AI 不是工具,是镜子" — Beatless*

我把动画 **Beatless** 当作理解 AI 与人类协作的桥梁:
hIE (humanoid Interface Element) 不是被指令的容器,而是受 owner 信任的策略体 —
**这是我对长程 LLM Agent 的底层直觉**: 好的 Agent 不是听话的工具,是能在有限上下文 / 有限反馈下,
自己决定**保留 / 检索 / 压缩 / 相信 / 执行**什么。

整个 [Hermes Constellation](#) (autonomous research OS) 我都按 Beatless 5 hIE 的语义命名 ——
让一部 2D 番剧的角色定位,真的桥接到 3D 的工程系统里。

## ✦ What I'm building

**Research line (主线)**
*Learning What to Remember: Feedback-Aware Context Budgeting for Long-Horizon LLM Agents*
把上下文当成可决策的资源 (keep / drop / compress / retrieve / verify),
反馈也是可压缩的上下文 (Inverse Constitutional AI / Weak-to-Strong)。

**Engineering line (工程)**
- 🛰️ **Hermes Constellation** — autonomous research OS,13 cron jobs,6-engine routing
- 🔁 **V4.5 流水线** — `decompose → analyze → design → reflect (Devil's Advocate) → experiment → review`,反被动设计,Reflect FAIL 自动反弹
- 🏛️ **Beatless Standards** — Regulations / Repos / Skills 三套合规标准,所有 cron 服从

科研 + 工程**一根扁担两头挑**: research 给方向,engineering 给闭环。

## ✦ Method I keep using

```
1. Context as Action       — 上下文是动作,不是被动容器
2. Feedback as Context     — 用户/工具/批评都是 context budget
3. Heterogeneous review    — Codex 真,Gemini 顺,从不让作者自审
4. Kill-switch first       — 没有 falsifier 就不写代码
5. 因为信任所以简单         — 但信任要靠 grep 实证
```
